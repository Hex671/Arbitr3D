# 🚀 Arbitr3D: 基于 MLLM 逻辑仲裁的零样本 3D 部件分割

Arbitr3D 是一个免训练、零样本（Zero-shot）的 3D 点云部件分割管线。它创新性地结合了 2D 基础模型（SAM + GLIP）的爆发式提取能力、3D 物理空间硬隔离算法，以及多模态大语言模型（MLLM）的逻辑常识仲裁能力，实现高 IoU 与高语义准确率的完美结合。

## 📁 目录结构

```text
Arbitr3D/
├── config/                 # 核心配置 (config.yaml)
├── data_prep/              # 点云数据加载与归一化
├── module_render/          # 3D 多视角机位生成与渲染 (RGB + Z-buffer)
├── module_2d_fm/           # 2D 基础模型特征提取 (GLIP 检测 + SAM 分割)
├── module_3d_lift/         # 2D-3D 反投影、宏观 IoU 合并与微观冲突硬隔离
├── module_mllm/            # MLLM 交互、Set-of-Mark 渲染与结果解析
├── evaluation/             # 分割指标评估与 Open3D 可视化
├── datatypes.py            # 全局数据结构 (Entity3D)
├── pipeline.py             # 端到端主控脚本
└── requirements.txt        # Python 依赖清单
```

## 🛠️ 环境安装

1. **基础依赖安装**
   请在终端中进入 `Arbitr3D` 目录并运行：
   ```bash
   pip install -r requirements.txt
   ```

2. **核心深度学习库（需根据您的 CUDA 版本手动配置）**
   - **PyTorch & torchvision**
   - **PyTorch3D** (推荐用于 `module_render` 渲染 Z-buffer)
   - **Segment Anything (SAM)**: `pip install git+https://github.com/facebookresearch/segment-anything.git`
   - **GLIP**: 参考 [GLIP 官方仓库](https://github.com/microsoft/GLIP) 进行编译和安装。

## ⚙️ 核心配置 (config.yaml)

在运行代码前，请检查 `config/config.yaml` 文件：
- **`render.num_cameras`**: 控制渲染的视角数量，默认为 20。
- **`model.glip_weight_path`** / **`sam_weight_path`**: 请替换为您本地下载的模型权重绝对路径。
- **`algorithm.iou_threshold`**: 3D 点簇的宏观合并交并比阈值（默认 `0.3`）。
- **`mllm.api_key_env`**: OpenAI 或其他 MLLM 的环境变量名称（如 `OPENAI_API_KEY`）。

在运行前，请务必在终端中设置您的 API 密钥：
```bash
# Windows (PowerShell)
$env:OPENAI_API_KEY="sk-您的API密钥"

# Linux / macOS
export OPENAI_API_KEY="sk-您的API密钥"
```

## 🚀 快速开始

可以使用核心流水线控制器 `pipeline.py` 一键拉起完整任务。以下是调用示例：

```python
from pipeline import Arbitr3DPipeline

# 1. 实例化 Pipeline，自动读取配置文件
pipeline = Arbitr3DPipeline(config_path="config/config.yaml")

# 2. 准备点云路径与 Prompt
point_cloud_path = "data/test_chair.ply" 
prompt_classes = "chair back, chair seat, chair leg, chair arm"

# 3. 运行端到端管线
pcd, final_labels, class_mapping = pipeline.run(
    point_cloud_filename=point_cloud_path, 
    prompt_classes=prompt_classes
)

print(f"检测到的类别映射: {class_mapping}")

# 4. 可视化结果
pipeline.visualizer.visualize_3d_result(pcd, final_labels)
```
